# 手機地圖投票網頁

這是一個簡單的手機投票系統：

- 可自定義 3 個投票項目名稱
- 使用 Google Maps 顯示 3 個地點
- 點擊地圖標記進行投票
- 透過 Socket.IO 即時同步投票結果給所有使用者
- **新增**：後台管理系統，透過帳密登入來設定地點名稱與座標
- **新增**：Supabase PostgreSQL 資料庫整合，用於持久化儲存投票和地點資料

## 使用方式

### 1. 安裝相依套件

```bash
cd C:\Users\User\mobile-vote
npm install
```

### 2. 設置 Supabase 資料庫

1. 前往 [Supabase](https://supabase.com) 註冊帳號並建立新專案
2. 在專案設定中找到 Project URL 和 anon public key
3. 編輯 `.env` 檔案，填入你的 Supabase 憑證：

```env
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your_supabase_anon_key
```

4. 在 Supabase 專案的 SQL Editor 中執行以下 DDL 來建立資料表：

```sql
-- 1. 基礎資料
CREATE TABLE basic_data (
    title_name TEXT,
    important_info TEXT
);

-- 2. 使用者資訊
CREATE TABLE user_info (
    user_name TEXT PRIMARY KEY,
    user_game_id TEXT,
    user_location TEXT
);

-- 3. 活動主檔
CREATE TABLE event_main (
    event_id TEXT PRIMARY KEY,
    event_name TEXT NOT NULL,
    event_type TEXT,
    event_date DATE,
    event_time TIME,
    voting_due TIMESTAMPTZ
);

-- 4. 地點主檔
CREATE TABLE location_main (
    location_id TEXT PRIMARY KEY,
    location_name TEXT NOT NULL,
    location_type TEXT,
    longitude NUMERIC(10, 7),
    latitude NUMERIC(10, 7)
);

-- 5. 獎勵主檔
CREATE TABLE award_condition (
    condition_id TEXT PRIMARY KEY,
    condition_type TEXT,
    condition_value INTEGER,
    condition_desc TEXT
);

-- 6. 投票活動設定
CREATE TABLE voting_event (
    voting_id TEXT PRIMARY KEY,
    event_id TEXT NOT NULL,
    location_1 TEXT,
    location_2 TEXT,
    location_3 TEXT,
    award_condition_1 TEXT,
    award_condition_2 TEXT,
    award_condition_3 TEXT,
    voting_due TIMESTAMPTZ
);

-- 7. 獎勵回饋資訊
CREATE TABLE award_feedback (
    feedback_id TEXT PRIMARY KEY,
    feedback_user TEXT,
    feedback_award TEXT,
    feedback_date DATE,
    feedback_status TEXT
);

-- 7. 見面會資訊
CREATE TABLE meetup_event (
    event_id TEXT,
    location_id TEXT,
    event_date DATE,
    event_time TIME,
    event_name TEXT,
    event_type TEXT,
    location_name TEXT,
    PRIMARY KEY (event_id, location_id)
);

-- 8. 投票資訊
CREATE TABLE voting_info (
    serial_no BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, 
    event_id TEXT,
    user_name TEXT, 
    voting_date TIMESTAMPTZ DEFAULT NOW(),
    voting_location TEXT,
    voting_award TEXT
);
```

5. 插入預設資料：

```sql
-- 插入基礎資料
INSERT INTO basic_data (title_name, important_info) 
VALUES ('手機地圖投票系統', '歡迎參與投票活動！');

-- 插入範例活動
INSERT INTO event_main (event_id, event_name, event_type, event_date, event_time, voting_due) 
VALUES ('event_001', 'GO FEST 2026', 'festival', '2026-07-11', '09:00:00', '2026-07-10 23:59:59+08');

-- 插入範例地點
INSERT INTO location_main (location_id, location_name, location_type, longitude, latitude) VALUES
  ('loc_001', '台北大安公園', 'park', 121.5444, 25.0267),
  ('loc_002', '台中勤美誠品', 'venue', 120.6458, 24.1511),
  ('loc_003', '高雄夢時代', 'venue', 120.3011, 22.5950);
```

### 3. 設置 Google Maps API

在 `public/index.html` 裡把 `YOUR_GOOGLE_MAPS_API_KEY` 換成你的 Google Maps API 金鑰。

### 4. 啟動伺服器

```bash
npm start
```

### 5. 開啟瀏覽器

**投票頁面**
```text
http://localhost:3000
```

**後台管理** （帳密為 admin / Mickey7911;;--）
```text
http://localhost:3000/admin.html
```

## 後台功能

進入後台管理頁面後，可以：
- 修改 3 個地點的名稱
- 修改 3 個地點的經緯度（Google Maps 地圖會即時更新）
- 所有投票者將立即看到更新後的地點資訊

## 系統功能

### 投票系統
- **即時投票**：用戶輸入名稱選擇地點進行投票
- **地圖整合**：Google Maps 顯示所有投票地點
- **即時統計**：投票結果即時更新，所有用戶都能看到最新統計
- **活動管理**：支援多個活動，系統自動選擇進行中的活動

### 管理後台
- **活動管理**：建立、編輯、刪除投票活動
- **地點管理**：管理投票地點的資訊和座標
- **統計查看**：查看每個地點的投票統計和獎勵分佈
- **即時同步**：管理員修改會即時同步到所有用戶

### 資料持久化
- **Supabase 整合**：所有投票資料自動儲存到雲端資料庫
- **活動歷史**：保留所有活動的投票記錄
- **用戶追蹤**：記錄每個用戶的投票行為
- **統計分析**：支援獎勵條件和回饋系統

## 資料表說明

- **basic_data**：系統基本資訊（標題、重要通知）
- **user_info**：用戶基本資料（遊戲ID、地點等）
- **event_main**：活動基本資訊（名稱、日期、投票截止時間）
- **location_main**：地點基本資訊（名稱、座標、類型）
- **award_condition**：獎勵條件設定
- **voting_event**：活動投票設定（包含 3 個投票地點與 3 個獎勵條件）
- **award_feedback**：獎勵發放記錄
- **meetup_event**：見面會活動資訊
- **voting_info**：投票記錄（包含用戶、時間、地點、獎勵）
